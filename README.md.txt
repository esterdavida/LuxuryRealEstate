Luxury Real Estate - מערכת תיווך דירות
פרויקט הכנה לפרקטיקום: Angular + .NET API + SQL Server
נושא הפרויקט:
מערכת לניהול ותצוגת נכסי נדל"ן יוקרתיים, המאפשרת שליפה, חיפוש, הוספה ועריכה של נכסים מול בסיס נתונים.
צד הלקוח:
Service מרכזי אחד: כל התקשורת מתבצעת דרך PropertyService מול ה-Endpoint היחיד  api/exec
Reactive Forms: השתמשנו ב-FormGroup לניהול טפסים וולידציות במסכי היצירה והעריכה
3 מסכים עיקריים: רשימת נכסים (כולל חיפוש מתקדם לפי תווך מחירים ועיר או סוג נכס), מסך פרטים מלאים, ומסך עריכה/הוספה.
מבנה בסיס הנתונים (DB)
הפרויקט מבוסס על 3 טבלאות:
Properties (טבלה ראשית)
PropertyTypes (סוגי נכסים כמו וילה/דירה).
Statuses (סטטוסים כמו למכירה/הושכר/חדש).
מבנה בסיס הנתונים וקשרים (Database Schema)
הפרויקט בנוי על מסד נתונים יחסי עם הקשרים הבאים:
טבלת Properties (ראשית): מכילה את כל פרטי הנכס.
טבלת PropertyTypes (עזר): מקושרת לטבלה הראשית בקשר של יחיד לרבים (1:N). שדה TypeId בטבלת הנכסים מפנה ל-Id בטבלה זו.
טבלת Statuses (עזר): מקושרת לטבלה הראשית בקשר של יחיד לרבים (1:N). שדה StatusId בטבלת הנכסים מפנה ל-Id בטבלה זו.

הסבר: הקשרים מבוצעים באמצעות Foreign Keys, מה שמבטיח שלא ניתן להזין נכס עם סטטוס או סוג שאינם קיימים בטבלאות העזר
Stored Procedures (SP)
העבודה מול ה-DB מתבצעת אך ורק דרך 5 פרוצדורות (Stored Procedures): Properties_GetAll, Properties_GetById, Properties_Create, Properties_Update, Properties_Search.
איך מריצים את הפרויקט?
DB: הרצתי את הסקריפטים בתיקיית db/ (יצירת טבלאות ופרוצדורות).
API: פתחי את הפרויקט ב-Visual Studio שם יש לי 3 עמודים התוכנית עצמה הגיסון והקונטרולים מריצים בוי הירוק נפתח גם מסך הפלט השחור וגם עמוד בקשות בדפדפן ואז אני יודעת שהשרת שלי חי 
Client: הריצו npm install ולאחר מכן ng serve בתיקיית client בשביל לפתוח אותו אחרי משיכה מהריפוזיטרי
